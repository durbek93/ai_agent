# 🤖 ИИ-Агент: Аналитик 80/20 (YouTube & Telegram Bot)

Этот проект представляет собой мощного автоматизированного ИИ-агента, который скачивает видео с YouTube (включая Shorts и Reels), распознает речь, вытягивает самую ценную информацию по принципу Парето (80/20) с помощью нейросетей Google Gemini и присылает готовую выжимку вместе с озвучкой в Telegram.

---

## 🚀 Основной функционал (Pipeline)

Проект работает как конвейер, состоящий из 5 ключевых этапов:
1. **📥 Скачивание**: `yt-dlp` скачивает видео/аудио по ссылке YouTube.
2. **🎧 Распознавание речи (STT)**: Модель `Whisper` (от OpenAI) переводит голос из видео в текст.
3. **🤖 Аналитика 80/20**: `Gemini 2.5 Flash` анализирует текст с помощью специального системного промпта, отсекая всю «воду» и оставляя только 20% самой важной сути, генерирующей 80% пользы.
4. **🎙️ Озвучка (TTS)**: `edge-tts` берет полученное текстовое саммари и озвучивает его приятным голосом.
5. **📱 Доставка**: Telegram-бот отправляет итоговый текстовый отчет и аудиофайл пользователю в чат.

## 🛠 Установка и настройка

### 1. Системные требования
- **Python 3.10+** (или выше)
- **FFmpeg** (системный пакет, необходим для обработки аудио в Whisper и yt-dlp)
- Установленный **Docker** (если планируется запуск в контейнере)

### 2. Клонирование и установка зависимостей
```bash
git clone <url_вашего_репозитория>
cd ai_agent
python3 -m venv .venv
source .venv/bin/activate  # Для Linux/macOS
# или .venv\Scripts\activate для Windows
pip install -r requirements.txt
```

### 3. Настройка переменных окружения
В проекте предусмотрен шаблон настроек:
1. Скопируйте файл `.env.example` и переименуйте его в `.env`.
2. Вставьте свои реальные ключи в `.env`:
   - `GEMINI_API_KEY`: Получить в [Google AI Studio](https://aistudio.google.com/).
   - `TELEGRAM_TOKEN`: Получить у [@BotFather](https://t.me/BotFather) в Telegram.

## 💻 Как использовать

### Запуск Telegram-бота
Бот будет слушать входящие сообщения. Просто отправьте ему ссылку на YouTube:
```bash
python main.py
```

### Локальная пакетная обработка (без Telegram)
Если нужно обработать список ссылок локально в консоли:
```bash
python localmain.py
```

### Запуск через Docker 🐳
Для полностью изолированного запуска (с пробросом папок наружу):
```bash
bash start.sh
```

## 📂 Структура проекта

* `main.py` — Главный файл с Telegram-ботом и полным циклом обработки в один проход (включает скачивание, Whisper STT, Gemini аналитику, TTS озвучку и очистку мусора).
* `localmain.py` — Консольная версия скрипта для пакетной обработки массива ссылок локально. Включает умный фильтр текста (удаление спецсимволов `*` и `#`) для чистого TTS.
* `download.py`, `transcribe.py`, `summarize.py` — Вспомогательные/тестовые модули (основная логика теперь интегрирована напрямую в `main.py` и `localmain.py`).
* `build.sh` — Скрипт для сборки Docker-образа (`ai agent`).
* `start.sh` — Скрипт для запуска Docker-контейнера в фоне с пробросом папок.
* `delete.sh` — Утилита для быстрой очистки сгенерированных артефактов (текстов и результатов).

## 🆕 Последние изменения (Changelog)

* **Монолитная архитектура**: Вся бизнес-логика (от скачивания до озвучки) объединена внутри `main.py` и `localmain.py` для большей надежности.
* **Умный фильтр для диктора**: Добавлена очистка текста от markdown-разметки (звездочек и решеток) перед передачей в `edge-tts`, при этом цифры сохраняются.
* **Оптимизация Gemini**: Использование стабильной модели `gemini-2.5-flash` для обхода лимитов и обработки больших объемов текста.
* **Скрипты автоматизации**: Добавлены bash-скрипты `build.sh` и `delete.sh` для удобного управления контейнерами и очистки пространства.
* **Примеры работы**: Добавлена папка `about_notion` с тестовыми данными (транскрипт и готовое саммари).

## 🔒 Безопасность и подготовка к GitHub

Проект был проанализирован и подготовлен к безопасной публикации в Open Source:

* **Добавлен `.gitignore`**: Из системы контроля версий строго исключены файлы конфигурации (`.env`), файлы сессий/cookies (`cookies.txt`), виртуальное окружение (`.venv`) и все генерируемые папки с данными (`downloads`, `transcripts`, `results`, `cache`). Это защищает приватные ключи API от утечек.
* **Добавлен `.env.example`**: Создан безопасный шаблон для конфигурации, чтобы новым разработчикам было легко развернуть проект.
* **Обновлен `.dockerignore`**: Файл `.env` исключен из контекста сборки Docker (`COPY . .`). Это предотвращает случайное "запекание" секретов внутрь Docker-образа, что делает контейнер абсолютно безопасным для публикации в Docker Hub или других registry.
