# 🤖 ИИ-Агент: Аналитик 80/20 (YouTube & Telegram Bot)

Этот проект представляет собой мощного автоматизированного ИИ-агента, который скачивает видео с YouTube (включая Shorts и Reels), распознает речь, вытягивает самую ценную информацию по принципу Парето (80/20) с помощью нейросетей Google Gemini и присылает готовую выжимку вместе с озвучкой в Telegram.

---

## 🚀 Основной функционал (Pipeline)

Проект работает как конвейер, состоящий из 5 ключевых этапов:
1. **📥 Скачивание**: `yt-dlp` скачивает видео/аудио по ссылке YouTube.
2. **🎧 Распознавание речи (STT)**: В классической версии (`main.py`) модель `Whisper` переводит голос в текст. В современной версии (`secondmain.py`) этот шаг пропущен — аудио скачивается в формате MP3 и отправляется сразу в Google Gemini API.
3. **🤖 Аналитика 80/20**: `Gemini` (версии 2.0/2.5 Flash) анализирует материал с помощью системного промпта, отсекая всю «воду» и оставляя только 20% сути, дающей 80% пользы.
4. **🎙️ Озвучка (TTS)**: `edge-tts` берет полученное текстовое саммари, безопасно фильтрует спецсимволы и озвучивает его.
5. **📱 Доставка**: Telegram-бот отправляет итоговый текстовый отчет и аудиофайл пользователю в чат.

## 🛠 Установка и настройка

### 1. Системные требования
- **Python 3.10+** (или выше)
- **FFmpeg** (системный пакет, необходим для обработки аудио в Whisper и yt-dlp)
- Установленный **Docker** (если планируется запуск в контейнере)

### 2. Клонирование и установка зависимостей
```bash
git clone <url_вашего_репозитория>
cd ai_agent
python3 -m venv .venv
source .venv/bin/activate  # Для Linux/macOS
# или .venv\Scripts\activate для Windows
pip install -r requirements.txt
```

### 3. Настройка переменных окружения
В проекте предусмотрен шаблон настроек:
1. Скопируйте файл `.env.example` и переименуйте его в `.env`.
2. Вставьте свои реальные ключи в `.env`:
   - `GEMINI_API_KEY`: Получить в [Google AI Studio](https://aistudio.google.com/).
   - `TELEGRAM_TOKEN`: Получить у [@BotFather](https://t.me/BotFather) в Telegram.

## 💻 Как использовать

### Запуск Telegram-бота
Бот будет слушать входящие сообщения. Просто отправьте ему ссылку на YouTube:
```bash
python main.py
```

### Локальная пакетная обработка (без Telegram)
Если нужно обработать список ссылок локально в консоли:
```bash
python localmain.py
```

### Запуск через Docker 🐳
Для полностью изолированного запуска (с пробросом папок наружу):
```bash
bash start.sh
```

## 📂 Структура проекта

* `main.py` — Классический файл Telegram-бота с полным циклом обработки (использует локальный Whisper для перевода в текст).
* `secondmain.py` — **Новый** Telegram-бот: сверхлегкая версия, которая отправляет MP3-аудио напрямую в облако Gemini (полностью исключая тяжелый Whisper).
* `localmain.py` — Консольная версия скрипта для пакетной обработки массива ссылок.
* `script_a_audio.py` — Отдельный скрипт: прямой анализ аудио через облачный API Gemini без использования STT.
* `script_b_subtitles.py` — Отдельный ультра-быстрый скрипт: мгновенно вытягивает субтитры через `youtube-transcript-api` и отдает их ИИ (вообще не качает медиа).
* `download.py`, `transcribe.py`, `summarize.py` — Вспомогательные/устаревшие модули.
* `build.sh`, `start.sh`, `delete.sh` — Bash-скрипты автоматизации для Docker и очистки кэша.

## 🆕 Последние изменения (Changelog)

* **Профессиональный UX бота**: Бот теперь динамически обновляет свое сообщение в реальном времени, показывая текущий этап работы (скачивание -> анализ -> озвучка). Также генерируемые файлы теперь получают осмысленные, очищенные имена на основе реальных названий видео с YouTube, вместо абстрактных временных меток.
* **Революция Cloud Audio**: Внедрены скрипты `secondmain.py` и `script_a_audio.py`. Теперь бот умеет скачивать MP3 и напрямую скармливать его облаку Gemini. Отказ от Whisper снизил требования к серверу до минимума.
* **Анализ по субтитрам**: Добавлен `script_b_subtitles.py` для анализа видеороликов через встроенные субтитры (молниеносный метод без загрузки медиа).
* **Надежность синхронизации с облаком**: В новых скриптах реализован Polling (безопасное ожидание статуса `PROCESSING` файла в облаке Google) и защита от утечки памяти (блок `finally`, гарантированно удаляющий файл из API Google после анализа).
* **Безопасный TTS-конвейер**: Текст перед передачей в диктор (`edge-tts`) сохраняется во временный файл (`-f`), исключая уязвимости (Command Injection) и проблемы с переносами строк.
* **Скрипты автоматизации**: Добавлены bash-скрипты для сборки Docker-образа и ручной очистки диска.

## 🔒 Безопасность и подготовка к GitHub

Проект был проанализирован и подготовлен к безопасной публикации в Open Source:

* **Добавлен `.gitignore`**: Из системы контроля версий строго исключены файлы конфигурации (`.env`), файлы сессий/cookies (`cookies.txt`), виртуальное окружение (`.venv`) и все генерируемые папки с данными (`downloads`, `transcripts`, `results`, `cache`). Это защищает приватные ключи API от утечек.
* **Добавлен `.env.example`**: Создан безопасный шаблон для конфигурации, чтобы новым разработчикам было легко развернуть проект.
* **Обновлен `.dockerignore`**: Файл `.env` исключен из контекста сборки Docker (`COPY . .`). Это предотвращает случайное "запекание" секретов внутрь Docker-образа, что делает контейнер абсолютно безопасным для публикации в Docker Hub или других registry.
